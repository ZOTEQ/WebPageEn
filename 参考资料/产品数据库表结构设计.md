# 产品详情页数据库表结构设计

## 一、需求分析

根据附图，产品详情页包含以下四个主要部分：

| 章节 | 内容类型 | 字段数量 | 说明 |
|------|----------|----------|------|
| 第一节 | 产品基础信息 | 固定 | 产品名、描述、CAS、FEMA、分子式、分子量、结构式 |
| 第二节 | 产品信息 | 固定7项 | Code、Synonym、Odor、Application、Shelf Life、Package、Storage |
| 第三节 | 附加说明1 | 可变 | 如Flash Point等 |
| 第四节 | 规格说明 | 可变 | 如Purity、Appearance、Solubility等 |

---

## 二、表结构设计

### 2.1 主表：products（产品基础信息）

| 字段名 | 类型 | 长度 | 约束 | 说明 |
|--------|------|------|------|------|
| id | INT | - | PRIMARY KEY, AUTO_INCREMENT | 产品ID |
| product_name | VARCHAR | 255 | NOT NULL | 产品名称 |
| product_description | TEXT | - | NULL | 产品描述 |
| cas_number | VARCHAR | 50 | NULL | CAS编号 |
| fema_number | VARCHAR | 50 | NULL | FEMA编号 |
| molecular_formula | VARCHAR | 100 | NULL | 分子式 |
| molecular_weight | DECIMAL | 10,4 | NULL | 分子量 |
| structural_formula | TEXT | - | NULL | 化学结构式（SVG/字符串） |
| category | VARCHAR | 100 | NULL | 产品类别（BIOBASED/SYNTHETIC/EU NAT/US NAT） |
| product_group | VARCHAR | 50 | NULL | 产品组号（如AR-001） |
| reach_status | BOOLEAN | - | DEFAULT FALSE | REACH注册状态 |
| created_at | DATETIME | - | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 2.2 子表：product_info（固定产品信息 - 第二节）

| 字段名 | 类型 | 长度 | 约束 | 说明 |
|--------|------|------|------|------|
| id | INT | - | PRIMARY KEY, AUTO_INCREMENT | ID |
| product_id | INT | - | FOREIGN KEY → products(id) | 关联产品ID |
| code | VARCHAR | 50 | NULL | 产品代码 |
| synonym | VARCHAR | 255 | NULL | 同义词 |
| odor_description | TEXT | - | NULL | 香气描述 |
| application | TEXT | - | NULL | 应用领域 |
| shelf_life | VARCHAR | 100 | NULL | 保质期 |
| package | VARCHAR | 255 | NULL | 包装规格 |
| storage_condition | VARCHAR | 255 | NULL | 储存条件 |

### 2.3 子表：product_additional_info（附加说明 - 第三节）

| 字段名 | 类型 | 长度 | 约束 | 说明 |
|--------|------|------|------|------|
| id | INT | - | PRIMARY KEY, AUTO_INCREMENT | ID |
| product_id | INT | - | FOREIGN KEY → products(id) | 关联产品ID |
| field_name | VARCHAR | 100 | NOT NULL | 字段名称（如Flash Point） |
| field_value | VARCHAR | 255 | NULL | 字段值（如109） |
| sort_order | INT | - | DEFAULT 0 | 排序顺序 |

### 2.4 子表：product_specification（规格说明 - 第四节）

| 字段名 | 类型 | 长度 | 约束 | 说明 |
|--------|------|------|------|------|
| id | INT | - | PRIMARY KEY, AUTO_INCREMENT | ID |
| product_id | INT | - | FOREIGN KEY → products(id) | 关联产品ID |
| spec_name | VARCHAR | 100 | NOT NULL | 规格名称（如Purity by GC） |
| spec_value | VARCHAR | 255 | NULL | 规格值（如98-100） |
| sort_order | INT | - | DEFAULT 0 | 排序顺序 |

---

## 三、ER关系图

```
products (1) ────────┬────────── (1) product_info
                     │
                     ├─────────── (N) product_additional_info
                     │
                     └─────────── (N) product_specification
```

- **products 1 : 1 product_info** - 一个产品对应一条固定信息
- **products 1 : N product_additional_info** - 一个产品可有多条附加说明
- **products 1 : N product_specification** - 一个产品可有多条规格说明

## 五、设计优势

1. **灵活性**：第三节和第四节使用动态键值对存储，支持任意数量的附加字段
2. **扩展性**：新增规格项无需修改表结构
3. **规范性**：第二节固定字段单独存储，便于查询和统计
4. **关联性**：通过外键关联，数据一致性有保障
5. **可排序**：附加字段支持自定义排序顺序
