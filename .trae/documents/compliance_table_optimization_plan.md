# Compliance & Documentation 表格优化方案

## 一、现状分析

### 当前表格结构
- **左卡片**：Legislatory Info（5行）+ Transportation Info（4行）
- **右卡片**：Support Documents（5行文档下载）
- **样式特点**：
  - 使用 `.compliance-table` 网格布局
  - 表头行：浅灰背景 + 左侧青色竖条装饰
  - 数据行：奇数行微青底纹，hover 时白底 + 阴影
  - 状态胶囊：青色（ok）/ 橙色（warn），固定宽度 100px
  - 代码值：JetBrains Mono 等宽字体
  - 下载按钮：白底青边胶囊 → hover 变青底白字

### 存在的问题
1. **视觉层次不够清晰**：表头与数据行的区分度不足
2. **创新性不足**：布局较为传统，缺乏视觉亮点
3. **交互反馈单一**：hover 效果较为基础
4. **信息密度不均**：左右卡片内容量差异较大

---

## 二、优化方案（基于三个 Skill）

### 方案 A：极简优雅风格（Impeccable 导向）

**设计理念**：清晰的信息架构 + 统一的视觉语言

#### 核心改进
1. **表头强化**
   - 表头行使用渐变背景：`linear-gradient(90deg, var(--ocean) 0%, var(--ocean-light) 100%)`
   - 文字改为白色，增强对比度
   - 移除左侧竖条装饰，改用整体渐变

2. **数据行优化**
   - 移除奇偶行背景色差异，统一使用纯白背景
   - 增加行间细分隔线：`border-bottom: 1px solid rgba(61, 139, 139, 0.1)`
   - hover 时：左侧出现 3px 青色竖条 + 轻微阴影

3. **状态胶囊升级**
   - 增加图标：Available → ✓ 图标，Yes → ✓ 图标
   - 胶囊宽度自适应内容，最小宽度 80px
   - 增加微动效：hover 时轻微缩放（1.05）

4. **代码值美化**
   - 增加浅青底纹 + 圆角背景
   - 字体大小提升至 0.85rem
   - 增加等宽字体图标前缀（如 `#` 符号）

#### 视觉效果
- 整体风格：简洁、专业、易读
- 信息层次：表头 > 子标题 > 数据行
- 交互反馈：细腻、不突兀

---

### 方案 B：创新视觉风格（Awwwards 导向）

**设计理念**：前沿技术 + 实验性布局 + 沉浸式体验

#### 核心改进
1. **卡片布局重构**
   - 左卡片拆分为两个独立卡片：Legislatory Info + Transportation Info
   - 三卡片使用瀑布流布局（masonry），高度自适应
   - 卡片圆角增至 24px，阴影加深

2. **表头视觉冲击**
   - 表头使用毛玻璃效果：`backdrop-filter: blur(10px)`
   - 背景半透明：`rgba(61, 139, 139, 0.15)`
   - 文字使用渐变色：`linear-gradient(135deg, var(--ocean) 0%, var(--sand) 100%)`

3. **数据行动效**
   - 入场动画：staggered fade-in（每行延迟 0.1s）
   - hover 时：整行向右平移 8px + 左侧出现青色光晕
   - 点击时：行背景闪烁一次（pulse 效果）

4. **状态胶囊创新**
   - 使用渐变背景：ok → 青色渐变，warn → 橙色渐变
   - 增加脉冲动画：`animation: pulse 2s infinite`
   - 胶囊形状改为圆角矩形（border-radius: 8px）

5. **下载按钮升级**
   - 改为图标按钮：仅显示下载图标，hover 时展开文字
   - 增加进度环动画（点击后）
   - 成功后显示 ✓ 图标 + "Downloaded" 文字

#### 视觉效果
- 整体风格：现代、炫酷、沉浸式
- 信息层次：动态呈现，引导视线
- 交互反馈：丰富、有趣、记忆点强

---

### 方案 C：平衡实用风格（Web-design 导向）

**设计理念**：规范先行 + 用户体验优先 + 可维护性

#### 核心改进
1. **表格结构优化**
   - 表头固定：使用 `position: sticky; top: 0`
   - 增加表格容器：`.compliance-table-wrapper`，支持滚动
   - 表头与数据行对齐优化：使用 `grid-template-columns: minmax(120px, 1.2fr) minmax(100px, 0.8fr)`

2. **视觉层次强化**
   - 表头背景：`var(--bg-secondary)` + 左侧 4px 青色竖条
   - 数据行：奇数行 `rgba(61, 139, 139, 0.05)`，偶数行纯白
   - hover 时：整行背景变为 `rgba(61, 139, 139, 0.1)` + 左侧竖条加深

3. **状态胶囊规范化**
   - 统一尺寸：`height: 28px; min-width: 90px; padding: 0 14px`
   - 增加图标：ok → ✓，warn → ⚠
   - 字体：Inter 600，大小 0.8rem
   - hover 时：轻微上浮（translateY(-2px)）+ 阴影

4. **代码值一致性**
   - 统一样式：浅青背景 + 等宽字体 + 圆角 6px
   - 字体大小：0.85rem
   - padding：4px 10px

5. **下载按钮优化**
   - 保持胶囊形状，增加图标 + 文字
   - hover 时：背景渐变 + 阴影上浮
   - 点击时：loading 动画（旋转图标）

6. **响应式优化**
   - 平板（768-1024px）：两列布局，卡片间距收紧
   - 手机（<768px）：单列布局，表头隐藏，标签/值纵向堆叠

#### 视觉效果
- 整体风格：专业、实用、易维护
- 信息层次：清晰、稳定、可预测
- 交互反馈：适度、不干扰阅读

## 三、推荐方案

### 推荐：方案 C（平衡实用风格）

**理由**：
1. **符合 DESIGN.md 规范**：严格遵循设计系统的颜色、字体、间距变量
2. **用户体验优先**：表头固定、滚动支持、响应式完善
3. **可维护性强**：样式规范、命名清晰、易于扩展
4. **性能优化**：避免过度动效，保证流畅度
5. **可访问性**：对比度合规、键盘导航友好

**实施优先级**：
1. **P0（必须）**：表格结构优化、视觉层次强化、响应式优化
2. **P1（重要）**：状态胶囊规范化、代码值一致性、下载按钮优化
3. **P2（可选）**：微动效增强、loading 动画

## 五、风险与应对

### 风险 1：表头固定可能影响滚动体验
**应对**：仅在表格高度超过容器时启用 sticky，否则使用静态布局

### 风险 2：动效过多可能影响性能
**应对**：使用 CSS transform 和 opacity（GPU 加速），避免 layout 和 paint

### 风险 3：响应式断点可能与其他模块冲突
**应对**：使用模块级媒体查询，避免全局样式污染
