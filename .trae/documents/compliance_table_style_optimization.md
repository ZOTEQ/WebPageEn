# Compliance & Documentation 表格风格优化方案

## 一、当前状态分析

### 现有结构
- **左卡片**：Legislatory Info（5行）+ Transportation Info（4行），共两个子表格
- **右卡片**：Support Documents（5行），单个表格
- 布局：`grid-template-columns: 1fr 1fr` 双列卡片

### 现有问题
1. **视觉层次单一**：所有行使用相同样式，缺乏节奏感
2. **表头与数据区分度不足**：仅靠背景色区分，对比度偏低
3. **状态胶囊过小**：`height: 26px` 在大屏幕上显得局促
4. **行高过大**：`line-height: 1.7` 导致行间距过疏，浪费空间
5. **分割线过于规则**：所有行都有相同的底部边框，缺乏变化
6. **响应式断点粗糙**：仅有一个 1024px 断点，缺乏平板适配

---

## 二、优化目标

1. **增强视觉层次**：通过间距、颜色、字体粗细区分表头、数据行、分组
2. **提升信息密度**：减小行高，增加有效信息占比
3. **优化交互反馈**：更优雅的 hover 效果，减少视觉干扰
4. **保持设计一致性**：遵循 DESIGN.md 的色彩系统和间距规范

---

## 三、具体优化方案

### 3.1 表格结构优化

| 元素 | 当前 | 优化后 | 理由 |
|------|------|--------|------|
| 行高 | `min-height: 40px` | `min-height: 48px` | 增加点击区域，提升可读性 |
| 行间距 | `line-height: 1.7` | `line-height: 1.5` | 紧凑但不拥挤 |
| 单元格内边距 | `padding: *1.5 *2` | `padding: *1.5 *2.5` | 增加水平呼吸空间 |
| 表头 | 背景 + 左侧竖条 | 背景 + 底部 2px 青色线 | 更现代的表头样式 |

### 3.2 视觉层次优化

| 元素 | 当前 | 优化后 |
|------|------|--------|
| 表头背景 | `var(--bg-secondary)` | `rgba(61, 139, 139, 0.06)` |
| 表头文字 | `font-weight: 600` | `font-weight: 700; letter-spacing: 0.08em` |
| 数据行奇数 | `rgba(61, 139, 139, 0.03)` | `transparent` |
| 数据行偶数 | `transparent` | `rgba(61, 139, 139, 0.02)` |
| 分割线 | `1px solid var(--border-subtle)` | `1px solid rgba(61, 139, 139, 0.08)` |

### 3.3 状态胶囊优化

| 元素 | 当前 | 优化后 |
|------|------|--------|
| 尺寸 | `height: 26px; min-width: 100px` | `height: 30px; min-width: 110px` |
| 字号 | `0.75rem` | `0.8rem` |
| 图标 | 无 | 添加 SVG 图标（✓/⚠） |
| 背景 | 纯色 | 渐变 `linear-gradient(135deg, ...)` |

### 3.4 代码值优化

| 元素 | 当前 | 优化后 |
|------|------|--------|
| 背景 | `rgba(61, 139, 139, 0.08)` | `transparent` |
| 边框 | 无 | `1px solid rgba(61, 139, 139, 0.15)` |
| 字体 | JetBrains Mono | 保留，增加 `font-variant-numeric: tabular-nums` |

### 3.5 下载按钮优化

| 元素 | 当前 | 优化后 |
|------|------|--------|
| 样式 | 边框胶囊 | 填充胶囊 + 图标 |
| hover | 背景变青 | 背景变深 + 上浮 + 阴影 |
| 图标 | 13px | 14px |

### 3.6 响应式优化

| 断点 | 当前 | 优化后 |
|------|------|--------|
| `≥1100px` | 双列 | 保留 |
| `768-1100px` | 双列收紧 | 双列 + 堆叠布局 |
| `<768px` | 单列 | 单列 + 表头隐藏 |

### 3.7 动效优化

| 元素 | 当前 | 优化后 |
|------|------|--------|
| 入场 | 无 | 淡入上滑（staggered） |
| hover 行 | 背景变化 + 阴影 | 左侧 3px 青色竖条 + 背景渐变 |
| hover 胶囊 | 无 | 轻微上浮 + 发光 |
| hover 下载 | 背景变色 | 背景渐变 + 图标旋转 |

## 五、风险控制

1. **保持现有功能**：不修改链接、数据、ARIA 标签
2. **保持设计一致性**：遵循 DESIGN.md 的色彩和间距系统
3. **渐进式修改**：先修改 CSS，再修改 HTML，避免一次性大改
4. **备份策略**：修改前备份，修改后验证，出问题可快速回滚
